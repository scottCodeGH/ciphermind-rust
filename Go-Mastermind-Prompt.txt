Create a smart and fun terminal-based puzzle game using Go. Below are the detailed requirements for this prompt:
	•	Game Concept: A code-breaking logic puzzle inspired by Mastermind. The program will think of a secret code, and the player has to guess it within a limited number of tries. Mastermind is a classic puzzle game where the aim is to find a hidden code through logic and deduction ￼. In this text version, use a sequence of symbols (e.g. 4 digits or 4 colored letters) as the secret code. After each guess, the game provides feedback: for example, how many symbols are correct in both value and position, and how many are correct but in the wrong position. The player uses these clues to refine their next guess. The goal is to crack the code before running out of attempts.
	•	Gameplay Interaction: The game runs in a loop reading the player's guesses from the terminal and printing clues. Make the interface user-friendly and fun despite being text-only: use clear prompts for input and well-formatted output for feedback. For instance, the program could output something like "Guess 3: YOURGUESS -> 1 symbol correct, 2 symbols correct but misplaced" after a guess. Include a bit of flavor or encouragement in text responses to keep it engaging (e.g. "Close call!" or "Try different combinations."). The puzzle should be solvable in a short session – perhaps allow around 8–10 guesses before game over, which typically takes a few minutes. Upon winning or losing, display a brief outcome message (e.g. "Congratulations, you cracked the code!" or "Game over, the code was ABCD.").
	•	Smart Challenge: Emphasize the "smart" aspect by ensuring the puzzle truly tests logic. The code should be generated randomly each game to offer a fresh challenge every time (like a new puzzle). This game will make players feel accomplished for deducing the secret, appealing to the casual puzzle solver who enjoys a quick brain-teaser. If Mastermind isn't used, an alternative can be a terminal Sudoku or crossword mini-puzzle, but Mastermind-style guessing fits the short play well. (Notably, even Sudoku has been implemented in terminal form ￼, proving that complex puzzles can work in text mode.)
	•	Technical Implementation: Use Go's standard I/O or a library like termbox-go to handle terminal input and output smoothly. (Termbox-go is a cross-platform library for building text-based interfaces in Go ￼ – it can help with reading keystrokes and controlling cursor output, though for a simple turn-based puzzle, standard printing and reading lines may suffice.) Structure the program with clear loops for each guess and use robust error handling (e.g. validate that the player's input meets the expected format and length, and prompt again if not). Although text-only, consider using ANSI escape codes or a package to add a bit of color to the output (for example, color the feedback or the guessed code) to enhance readability and fun. Keep the code efficient and safe (leveraging Go's strengths in concurrency and simplicity), so the game never crashes and handles edge cases (like invalid input) gracefully. The result should be a quick, brain-teasing game that runs in a terminal and showcases Go's ability to create enjoyable experiences even without graphics ￼.
